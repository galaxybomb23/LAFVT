# Fix Suggester

The Fix Suggester is a standalone tool in the LAFVT pipeline that generates potential code fixes for bugs identified during the formal verification and review stages. It queries a Large Language Model (LLM) to propose code changes based on the violation assessments.

NOTE: This module is currently under development and is not yet integrated into the main LAFVT pipeline.

## Overview

The `fix_suggester.py` script parses the `violation_assessments.json` file generated by LAFVT. For each confirmed bug (based on a minimum threat score input by the user), it extracts the target function's signature and the bug context around the crash site, and then prompts an LLM to generate a `diff`-style code suggestion to resolve the issue.

The output is a structured JSON file (`fix_suggestions.json`) containing proposed fixes along with an explanation of why the fix resolves the violation.

## Usage

Run the Fix Suggester by pointing it at the assessment outputs directory and the original project source directory:

```bash
python src/fix_suggester.py \
    --output_dir <path/to/lafvt_output> \
    --project_dir <path/to/source_code_of_project> \
    [--llm_model <model_name>] \
    [--limit N] \
    [--min_threat N]
```

### Arguments

| Argument          | Required | Default         | Description                                                                              |
|-------------------|----------|-----------------|------------------------------------------------------------------------------------------|
| `--output_dir`    | Yes      | —               | Directory containing `violation_assessments.json`.                                       |
| `--project_dir`   | Yes      | —               | Root directory of the verified C/C++ project (e.g., the `RIOT` repo folder).             |
| `--llm_model`     | No       | `llama4:latest` | The LLM model to use for generating fixes (runs against GenAI API by default).         |
| `--limit`         | No       | `None`          | Limit the number of suggestions to generate, which is useful for quick testing.          |
| `--min_threat`    | No       | `1`             | Minimum threat score (greater than or equal to this value) from the LLM Review to consider for generating a fix. The value range is from 1 to 10.              |
| `--GENAI_API_KEY` | No       | reads `.env`    | Override the GenAI backend API key (also falls back to `.env` or `OPENAI_API_KEY`).      |

NOTE: The `min_threat` parameter is used to filter out the bugs that are not severe enough to be fixed. The value range is from 1 to 10, where 1 is the least severe and 10 is the most severe. It ignores all false positives (thread score == -1).

NOTE: The fix suggestor is decoupled from AutoUP and works on Purdue RCAC GenAI Studio API instead of OpenAI API. Therefore, add a Purdue RCAC GenAI Studio API key to the `.env` file as `GENAI_API_KEY`.

## Output

The script creates a `fix_suggestions/` subdirectory within the provided `--output_dir`.

```text
<output_dir>/fix_suggestions/
├── fix_suggester.log       # Execution log for the suggestion process
└── fix_suggestions.json    # The generated fix suggestions
```

### `fix_suggestions.json` Format

Each entry in the output JSON contains:
- **Target Function**: The name of the function where the violation was found.
- **Source File**: The path to the original source file.
- **Violated Precondition**: The specific precondition or assertion that failed.
- **Fix Suggestion**: The LLM output parsed as JSON, including:
  - `is_fixable` (boolean): Whether the LLM determined the bug could be reasonably fixed given the context.
  - `explanation` (string): A brief explanation of why the proposed fix resolves the violation.
  - `suggested_code_diff` (string): The suggested code changes in standard diff format.
