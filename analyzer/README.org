* To run
- analyzer.py needs a compile_commands.json file which can be generated with gen_cc.py
- Example usage shown below
** gen_cc.py:
usage: gen_cc.py [-h] --riotbase RIOTBASE --board BOARD [-j JOBS] [--clean] [--bear BEAR] app_dir

Generate compile_commands.json for a RIOT app directory using bear.

positional arguments:
  app_dir              Path to RIOT app directory

options:
  -h, --help           show this help message and exit
  --riotbase RIOTBASE  Path to RIOT base directory
  --board BOARD        RIOT board (e.g., native)
  -j, --jobs JOBS      Parallel jobs for make
  --clean              Run make clean before building
  --bear BEAR          Path to bear binary (or name on PATH)
** analyzer.py:
usage: analyzer.py [-h] [--target TARGET] [--sysroot SYSROOT] target_dir

Analyze C/C++ functions defined in a CCDB(compile_commands.json) using LEOPARD-style metrics.

positional arguments:
  target_dir         Directory containing the compile_commands.json file

options:
  -h, --help         show this help message and exit
  --target TARGET    Clang target ABI (e.g., arm-none-eabi).
  --sysroot SYSROOT  Sysroot path for the target toolchain.
* Output
- Each row represents one function's metrics
- Each column corresponds to a specific complexity of vulnerability metric
* Metrics explanation
** Complexity metrics
*** C1: Cyclomatic complexity
- Number of linearly independent paths through a function
*** Loop structure metrics
- C2: Number of loops
- C3: Number of nested loops
- C4: Maximum nesting level
*** Binning strategy
- Computer complexity score by adding all complexity metric values, group functions with same score into the same bin
** Vulnerability metrics
*** Dependencies
- Dependency relationship of a function with other functions
- V1: Number of parameter variables
- V2: Number of variables prepared by the function as parameters of function calls
*** Pointer metrics
- Captures manipulation of pointers
- # of pointer arithmetic metrics can be obtained from the AST
- V3: # of pointer arithmetic operations
- V4: # of unique variables used in pointer arithmetic
- V5: Max pointer arithmetic a variable is involved in
*** Control structures
- V6: # of nested control structures
- V7: Max nesting level of control structures
- V8: Maximum of control structures that are control-dependent
- V9: Maximum of data-dependent control structures
- V10: # of if statements without explicit else statement
- V11: # of variables involved in control predicates
